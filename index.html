Introduction

{{apiName}} contains deep fundamental information about most ETFs, ETNs, ETCs and other fund-like products listed on all major european exchanges.

{{apiName}} provides basic information such as: Domicile, Followed Index, Asset Class, Fee, Currency, Replication Method, Distribution Type & Frequency. This information is manually analyzed by our team and updated regulary. Listings of new products are reflected almost immediately.

{{apiName}} also provide deep fundamental information derived from the underlying securities such as: Sector Exposre, Country Exposure, Factors, Dividend Yield etc. This information is dynamically updated as the underlying securities change or are added / removed from their tracking product.

Get Individual Product

This endpoint searches for a product (like ETFs) by its ISIN

Retrieves all of the information known for a specific product in a JSON format.

In order to search for a product with a specific ISIN write:

{{apiUrl}}/product/ISIN

Parameters

  • isin String Mandatory

    The ISIN that distinctly identifies the product.

Dynamic Search

This endpoint searches products (like ETFs) by dynamic criteria, and retrieves the results in a JSON format.

This endpoint is not meant as a way to extract coprehensive information about the desired products. It will return a list of basic attributes (ISIN & Name). Any further information needs to be fetched with using the Individual Endpoints

All of the parameters listed below can be attached as query parameters in an arbitary order in order to build a request:

{{apiUrl}}/search?param1=v1&param2=v2

GET{{apiUrl}}/search?exchange=SIX&USA=20&TECHNOLOGY=30

Parameters

exchange Exchange Optional

Will filter products only listed on a specific exchange. The supported exchanges are: LSE, XETRA, BORSA_ITALIANA, SIX

To get a list of the products listed on an exchange EX write:

{{apiUrl}}/search?exchange=EX

GET{{apiUrl}}/search?exchange=SIX
Returns a list of funds listed on SIX
factor Factor Optional

Returns products with the given factor. Factor definitions follow the MSCI Factors:

VOLATILITY_HIGH VOLATILITY_LOW YIELD QUALITY MOMENTUM VALUE SIZE_BIG SIZE_SMALL MULTIFACTOR

In order to search for a product with a specific factor F write:

{{apiUrl}}/search?factor=F

GET{{apiUrl}}/search?factor=MOMENTUM
Returns a list of products which follow a Momentum factor strategy.
[:ALPHA3-COUNTRY-CODE] Numeric Optional

The products will be filtered by exposure to the provided Alpha3-Country-Code. The minimum exposure required is the value for this parameter.

In order to search for products with at least p% exposure to country write:

{{apiUrl}}/search?country=P

GET{{apiUrl}}/search?IRL=45
Returns a list of funds which have at least 45% of their assets invested in Ireland
GET{{apiUrl}}/search?RUS=1&HUN=1&POL=1
Finds products with some broad eastern europe exposure.
[:SECTOR] Numeric Optional

The products will be filtered by exposure to the provided Sector. The minimum exposure required is the value for this parameter.

The sectors follow the Global Industry Classification Stadnard:

INFORMATION_TECHNOLOGY COMMUNICATION_SERVICES INDUSTRIALS HEALTH_CARE MATERIALS REAL_ESTATE ENERGY CONSUMER_DISCRETIONARY CONSUMER_STAPLES UTILITIES FINANCIALS

In order to search for products with at least p% exposure to a sector write:

{{apiUrl}}/search?sector=p

GET{{apiUrl}}/search?ENERGY=30&FINANCIALS=30

Returns products with at least 30% exposure to Energy and 30% exposure to Financials.

totalFee Numeric Optional

Will filter products that have at most the total fee provided.

To get a list of the products that have at most p% fee write:

{{apiUrl}}/search?totalFee=p

GET{{apiUrl}}/search?totalFee=4.5
Returns a list of products that have at most 4.5% fee. For instance all ETFs with TERS smaller or equal than 4.5 will be included here.
domicile ALPHA3-COUNTRY-CODE Optional

Returns products which are domiciles in the provided country. The country provided should be in a Alpha3-Country-Code format.

To get a list of the products that are domiciled in country CTY write:

{{apiUrl}}/search?domicile=CTY

GET{{apiUrl}}/search?domicile=LUX
Returns a list of products that are domiciled in Luxembourg
replicationMethod ReplicationMethod Optional

Returns products with the given replication method. The available replication methods are:

PHYSICALSYNTHETIC

In order to search for a product with replication method REPLICATION_METHOD write:

{{apiUrl}}/search?replicationMethod=REPLICATION_METHOD

GET{{apiUrl}}/search?replicationMethod=PHYSICAL
Returns a list of products which are physically
distributionType DistributionType Optional

Returns products with the given distribution type. The available distribution types are:

DISTRIBUTINGACCUMULATING

In order to search for a product with distribution type DIST_TYPE write:

{{apiUrl}}/search?distributionType=DIST_TYPE

GET{{apiUrl}}/search?distributionType=DISTRIBUTING
Returns a list of products which distribute their dividends.
assetClass AssetClass Optional

Returns products with the given asset class. The available asset classes are:

FIXED_INCOME COMMODITIES REAL_ESTATE EQUITY CURRENCY

In order to search for a product with asset class ASSET write:

{{apiUrl}}/search?assetClass=ASSET

GET{{apiUrl}}/search?assetClass=EQUITY
Returns a list of products with asset class EQUITY (e.g., ETFs and Equity Funds)
productType ProductType Optional

Returns products of the given type. The available product types are:

ETF ETN ETC FUND

While the support for ETFs is excelent, support for other product types is still experimental.

In order to search for a product from a type PT write:

{{apiUrl}}/search?productType=PT

GET{{apiUrl}}/search?productType=ETF
Returns a list of ETFs

Authentication

{{apiName}} uses tokens to authenticate requests. These tokens are computed automatically by {{apiName}} when the page is visited. Tokens are loaded with time and calls when you subscribe.

Once a token has been loaded, it can be renewed, cancelled and managed.

To make an request, just pass the token in the query string as follows:

https://api-url?token=YOUR_TOKEN

Your tokens carry many privileges, so be sure to keep them secure! Do not share your token in publicly accessible areas such as GitHub, client-side code, and so forth.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail. There is a daily limit of a few unauthenticated calls which can be used to test the API.
