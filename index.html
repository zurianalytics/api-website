About ETF Data API

We provide deep fundamental information, derived using AI, about most ETFs, ETNs, ETCs and other fund-like products listed on all major European exchanges. The free and base service is a JSON API.

Includes basic information such as: Domicile, Followed Index, Asset Class, Fee, Currency, Replication Method, Distribution Type & Frequency.

{{apiName}} also provides deep fundamental information derived using ML to learn about the underlying securities such as: Sector Exposure, Country Exposure, Factors, Dividend Yield etc.

Individual Product

This endpoint searches for a product (like ETFs) by its ISIN.

Retrieves all of the information known for a specific product in a JSON format.

In order to search for a product with a specific ISIN use:

{{apiUrl}}/product/ISIN

Parameters

  • isin String Mandatory

    The ISIN that distinctly identifies the product.

Dynamic Search

This endpoint searches products (like ETFs) by dynamic criteria, and retrieves the results in a JSON format

This endpoint is not meant as a way to extract comprehensive information about the desired products. It will return a list of basic attributes (ISIN & Name). Any further information needs to be fetched with using the Individual Endpoints

All of the parameters listed below can be attached as query parameters in an arbitrary order in order to build a request:

{{apiUrl}}/search?param1=v1&param2=v2

GET{{apiUrl}}/search?exchange=XETRA&DEU=20&INFORMATION_TECHNOLOGY=30
{"content":[{"name":"ComStage STOXX Europe 600 Technology UCITS ETF","isin":"LU0378437098"},{"name":"iShares STOXX Europe 600 Technology UCITS ETF (DE)","isin":"DE000A0H08Q4"},{"name":"Lyxor STOXX Europe 600 Technology UCITS ETF - Acc","isin":"LU1834988518"}],"metaInfo":{"pageNumber":0,"pageSize":10,"totalPages":1,"totalElements":3}}

Parameters

exchange Exchange Optional

Will filter products only listed on a specific exchange. The supported exchanges are: LSE XETRA BORSA_ITALIANA SIX

To get a list of the products listed on an exchange EX use:

{{apiUrl}}/search?exchange=EX

GET{{apiUrl}}/search?exchange=SIX
Returns a list of funds listed on SIX.
factor Factor Optional

Returns products with the given factor. Factor definitions follow the MSCI Factors:

VOLATILITY_HIGH VOLATILITY_LOW YIELD QUALITY MOMENTUM VALUE SIZE_BIG SIZE_SMALL MULTIFACTOR

In order to search for a product with a specific factor F use:

{{apiUrl}}/search?factor=F

GET{{apiUrl}}/search?factor=MOMENTUM
Returns a list of products which follow a Momentum factor strategy.
[:ALPHA3-COUNTRY-CODE] Numeric Optional

The products will be filtered by exposure to the provided Alpha3-Country-Code. The minimum exposure required is the value for this parameter.

In order to search for products with at least p% exposure to country use:

{{apiUrl}}/search?country=P

GET{{apiUrl}}/search?IRL=45
Returns a list of funds which have at least 45% of their assets invested in Ireland.
GET{{apiUrl}}/search?RUS=1&HUN=1&POL=1
Finds products with some broad Eastern Europe exposure.
[:SECTOR] Numeric Optional

The products will be filtered by exposure to the provided Sector. The minimum exposure required is the value for this parameter.

The sectors follow the Global Industry Classification Standard:

INFORMATION_TECHNOLOGY COMMUNICATION_SERVICES INDUSTRIALS HEALTH_CARE MATERIALS REAL_ESTATE ENERGY CONSUMER_DISCRETIONARY CONSUMER_STAPLES UTILITIES FINANCIALS

In order to search for products with at least p% exposure to a sector use:

{{apiUrl}}/search?sector=p

GET{{apiUrl}}/search?ENERGY=20&FINANCIALS=20

Returns products with at least 20% exposure to Energy and 20% exposure to Financials.

totalFee Numeric Optional

Will filter products that have at most the total fee provided.

To get a list of the products that have at most p% fee use:

{{apiUrl}}/search?totalFee=p

GET{{apiUrl}}/search?totalFee=4.5
Returns a list of products that have at most 4.5% fee. For instance all ETFs with TERS smaller than or equal to 4.5% will be included here.
domicile ALPHA3-COUNTRY-CODE Optional

Returns products which are domiciles in the provided country. The country provided should be in an Alpha3-Country-Code format.

To get a list of the products that are domiciled in country CTY use:

{{apiUrl}}/search?domicile=CTY

GET{{apiUrl}}/search?domicile=LUX
Returns a list of products that are domiciled in Luxembourg.
replicationMethod ReplicationMethod Optional

Returns products with the given replication method. The available replication methods are:

PHYSICAL SYNTHETIC

In order to search for a product with replication method REPLICATION_METHOD use:

{{apiUrl}}/search?replicationMethod=REPLICATION_METHOD

GET{{apiUrl}}/search?replicationMethod=PHYSICAL
Returns a list of products which are physically replicated.
distributionType DistributionType Optional

Returns products with the given distribution type. The available distribution types are:

DISTRIBUTING ACCUMULATING

In order to search for a product with distribution type DIST_TYPE use:

{{apiUrl}}/search?distributionType=DIST_TYPE

GET{{apiUrl}}/search?distributionType=DISTRIBUTING
Returns a list of products which distribute their dividends.
assetClass AssetClass Optional

Returns products with the given asset class. The available asset classes are:

FIXED_INCOME COMMODITIES REAL_ESTATE EQUITY CURRENCY

In order to search for a product with asset class ASSET use:

{{apiUrl}}/search?assetClass=ASSET

GET{{apiUrl}}/search?assetClass=EQUITY
Returns a list of products with asset class EQUITY (e.g. ETFs and Equity Funds)

Authentication

{{apiName}} uses tokens to authenticate requests. These tokens are computed automatically by {{apiName}} when the page is visited. Tokens are loaded with time and calls when you subscribe.

Once a token has been loaded, it can be managed, cancelled and renewed.

To make a request, just pass the token in the query string as follows:

https://api-url?token=YOUR_TOKEN

Your tokens carry many privileges, so make sure to keep them secure! Do not share your token in publicly accessible areas such as GitHub, client-side code, and so forth.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail. There is a daily limit of a few unauthenticated calls.
